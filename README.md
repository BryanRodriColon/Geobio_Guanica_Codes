# Geobio_Guanica_Codes

## Repository Overview

This repository contains the bioinformatic and statistical codes used in support of the manuscript entitled **“Documenting the Occurrence of Microbial Carbonate Deposits in Southwestern Puerto Rico”**, by **Bryan J. Rodríguez-Colón et al.** The manuscript has been submitted to *Scientific Reports* and is also available as a preprint on **EarthArXiv** under the same title.

The scripts archived here were used for processing, normalization, analysis, and visualization of 16S rRNA amplicon sequencing data, as well as for generating the figures and summary tables reported in the manuscript. This repository is intended to ensure transparency and reproducibility of the microbial ecology component of the study.

Raw sequencing data (FASTQ files) are not hosted in this repository and are instead publicly available through [Figshare](https://figshare.com/s/fd4e957a69e6de2036d4)

### *Terminology and Nomenclature Note*

In the code and data files provided in this repository, the abbreviations **PO**, **JB**, and **PS** are used to denote the three lagoon systems analyzed in this study (PO = Salinas Vernales, JB = Salinas Salinetas, PS = Laguna Providencia).
In the associated manuscript, these systems are referred to using a different set of abbreviations (**SV**, **SS**, and **LP**, respectively).
This difference reflects an update in nomenclature adopted during manuscript preparation and does not affect the underlying data structure or analyses.

## Terminal_Codes/

This directory contains shell and SLURM scripts used to preprocess 16S rRNA amplicon sequencing data in QIIME2 on an HPC system.

### Scripts (in execution order)

#### Code 1: `code1_qiime2_cutadapt_demuxAlready_AndFastQC.sh`
Performs primer and adapter trimming on demultiplexed paired-end reads using QIIME2 Cutadapt.  
**Key parameters:**  
- Forward adapter: `GGATTAGATACCCBDGTAGTC`  
- Forward primer: `CCTACGGGNGGCWGCAG`  
- Reverse adapter: `CTGCWGCCNCCCGTAGG`  
- Reverse primer: `GACTACHVGGGTATCTAATCC`

#### Code 2: `code2_qiime_dada2_code.slurm`
Runs QIIME2 DADA2 denoising on trimmed paired-end reads to infer ASVs.  
**Key parameters:**  
- Truncation lengths: forward = `240`, reverse = `190`  
- Trim-left: forward = `10`, reverse = `10`  
- Chimera method: `consensus`  
- Threads: `4`  
- Reads used for error model learning: `400000`

#### Code 3: `code3_qiime2_taxonomy_phylogeny_code.slurm`
Performs taxonomic assignment and phylogenetic reconstruction in QIIME2.  
**Key parameters:**  
- Taxonomy classification: `classify-sklearn` using a SILVA 138-based classifier  
- Alignment: MAFFT  
- Tree inference: FastTree  
- Rooting: midpoint-root


#### Additional codes: `Biom_manipulation`
Performs post-QIIME2 BIOM manipulation to prepare a taxonomy-annotated feature table for downstream R analyses. This workflow converts BIOM tables to TSV, filters taxa and samples based on confidence and domain-level criteria, standardizes taxonomic rank formatting, and merges updated taxonomy back into the feature table before reconversion to BIOM format. The resulting BIOM is structured for direct import into phyloseq.


## R_codes/

This directory contains R workflows for downstream analysis of 16S rRNA amplicon sequencing data, including normalization, construction of phyloseq objects, relative abundance summaries, plotting, and taxonomy table exports. Each script begins with custom helper functions used throughout the workflow.

### Folder structure

- `InputData/`: BIOM table(s), sample metadata, and phylogenetic tree  
- `OutputData/`: Saved R objects and statistical tables generated by the scripts
- `DataSummary/`: Saved excel files with taxonomic summary produced in Code 7.

### Scripts (in execution order)

#### Code 1: `code1_css-normalization.Rmd`
Applies cumulative sum scaling (CSS) normalization to sparse amplicon count data using `metagenomeSeq`. The workflow imports BIOM counts, filters non-target sequences (non-Bacteria/Archaea), handles empty samples to avoid normalization failures, and performs CSS as an alternative to rarefaction (Paulson et al., 2013).

#### Code 2: `Pre_processing_After_cssNorm.Rmd`
Imports BIOM table(s), parses SILVA-formatted taxonomy into standard ranks, loads metadata and a phylogenetic tree, and constructs phyloseq objects for downstream analysis. Data are also subset by domain (Bacteria and Archaea). A non-normalized phyloseq object is retained for statistical testing workflows.

#### Code 3: `RelAbund_UnclassifiedCleaning.Rmd`
Calculates relative abundance (%) tables from phyloseq objects and prepares cleaned long-format datasets for plotting and summary export. Summaries are generated across key grouping variables (e.g., sample layer, sample identity, and sample type) for All Prokaryotes, Bacteria-only, and Archaea-only.

#### Code 4: `16S_analysis_Plotting.Rmd`
Generates stacked bar plots of community composition across taxonomic ranks (Domain through Genus) for All Prokaryotes, Bacteria, and Archaea. The script uses rank-wise aggregation (top taxa with pooled “Other”), consistent sample ordering, and reproducible color mapping to generate manuscript-ready figures.

#### Code 5: `MicrobialGuilds_Plotting.Rmd`
Guild-based plotting workflow.

#### Code 6: `Statistical_Analysis.Rmd`
Statistical analysis workflow for Alpha and Beta Diversity and Venn Diagram Plotting. 

#### Code 7: `tax_summary.Rmd`
Exports taxonomy summary tables (Phylum through Genus) for All Prokaryotes, Bacteria-only, and Archaea-only across key grouping variables (e.g., sample layer, sample identity, and sample type).
